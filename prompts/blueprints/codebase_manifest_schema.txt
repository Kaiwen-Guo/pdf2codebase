Codebase manifest artifact schema:
{
  "entry_module": "safe_python_module_name",
  "files": [
    {
      "path": "relative/path.py",
      "kind": "source|support",
      "purpose": "short purpose",
      "content": "complete file content"
    }
  ]
}

Codebase manifest rules:
- Return only valid JSON matching this schema.
- Generate source/support files only; tests are generated by a separate stage.
- File paths must be relative, must not contain "..", and should be importable from the generated codebase root.
- Keep the codebase small and inspectable.
- Prefer one source module unless the project spec clearly requires multiple files.
- The entry_module must match the module that contains the primary public API.
- Source files should be placed at paths importable from the generated source root.
- Do not include generated tests in this manifest unless explicitly performing repair.
- Do not include shell scripts, notebooks, binary data, hidden files, absolute paths, or package manager files.
- File contents must be complete and runnable as written.
- Use docstrings/comments only when they clarify requirement mapping or non-obvious logic.
